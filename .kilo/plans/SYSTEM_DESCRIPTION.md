# Отчет по сегментации заказчиков корпоративных перевозок

## Назначение

Скрипты анализируют контент тендерных документов (ТЗ на перевозку сотрудников) из папки `source/`, извлекают структурированные признаки, классифицируют заказчиков по сегментам и генерируют HTML-отчёт.

## Быстрый старт

```bash
# 1. Установка зависимостей
pip install -r requirements.txt

# 2. Полный запуск (базовая статистика + контентный анализ + HTML-отчёт)
python3 -m scripts.main

# 3. Только контентный анализ (без базовой статистики)
python3 -m scripts.main --skip-basic

# 4. Только базовая статистика (без контентного анализа)
python3 -m scripts.main --skip-content
```

## Организация папки source/

**1 папка = 1 компания.** Все файлы одного заказчика кладутся в одну папку. Название папки становится названием компании в отчёте.

Если в папке несколько файлов — они схлопываются в один юнит для аналитики:
- Булевы признаки объединяются через OR (есть хотя бы в одном файле)
- Числовые берутся максимальные
- Сегмент — по большинству файлов

Файлы с нечитаемыми форматами (`.doc`, битые PDF, скриншоты) обрабатываются без ошибок — просто пропускаются.

## Как добавить новый файл

1. Положить файл в папку с названием компании в `source/`. Если папки нет — создать.
2. Если это новая компания — папку назвать как она должна отображаться в отчёте.
3. Запустить `python3 -m scripts.main`.
4. Открыть `results/report.html`.

## Как работает сегментация

### Извлечение признаков (extractor.py)

Из текста каждого документа извлекаются:

| Признак | Тип | Пример |
|---|---|---|
| ГЛОНАСС/мониторинг | boolean | есть/нет |
| Кондиционер | boolean | есть/нет |
| Медосмотр | boolean | есть/нет |
| Требования безопасности | boolean | есть/нет |
| Требования к водителю | boolean | есть/нет |
| Кол-во сотрудников | число | 240 |
| Кол-во маршрутов | число | 1 |
| Вместимость | число | 19 |

Также извлекаются строковые поля (компания, регион, номер тендера, срок контракта, график работы) — они не отображаются в таблицах признаков, но участвуют в подсчёте "есть хоть какие-то данные".

### Классификация по сегментам (segmenter.py)

Каждый документ относится к одному из 7 сегментов на основе:
1. **Ключевые слова** в имени папки/файла и в тексте (первые 3000 символов)
2. **Паттерны** в тексте (например, "круглосуточный", "строительство", "223-ФЗ")
3. **Извлечённые признаки** (наличие ГЛОНАСС, медосмотра и т.д.)
4. **Числовые характеристики** (кол-во сотрудников > 200 и т.д.)

Каждое совпадение даёт взвешенный балл. Пороговая логика:
- `best_score < 0.5` → "Не определено"
- `best_score >= 2.0` или разница между топ-1 и топ-2 > 0.5 → уверенный выбор
- Иначе → победитель по наибольшему баллу

### 7 сегментов

| Сегмент | Описание | Примеры |
|---|---|---|
| **Торговля/Производство** | Компании сферы торговли и производства продуктов питания. Доставка на производственные площадки и склады. | МАКФА, Метро, Фаберлик, Казанский ЖК, Экоальянс (АО ТВЭЛ), ИДС Боржоми, КПО Егорьевск |
| **Промышленность** | Крупные промышленные предприятия и заводы. Большое кол-во сотрудников, строгие требования к безопасности. | АвтоВАЗ, РМИ-Сталь, Трубная металлургическая компания, РН-Проектирование |
| **Строительство** | Строительные компании. Вахтовый метод, удалённые объекты. | Стройтранснефтегаз, Трансинжстрой, Капстрой |
| **IT/Услуги** | IT-компании и сервисные организации. Офисные сотрудники. | Ozon, Байон, Эрманн |
| **Госсектор** | Госкорпорации и унитарные предприятия. Закупки по 223-ФЗ. | Росатом, ФГУП по ОрВД, Россети НТЦ |
| **Логистика/Почта** | Почтовые и логистические операторы. Круглосуточная работа. | Почта России, СДЭК |
| **Транспорт/Перевозки** | Транспортные компании-подрядчики (сами являются предметом закупки). | Деловые Линии |

## Что генерируется в отчёте

1. **Карточки** — общее количество документов, сегментов, с извлечёнными признаками
2. **Распределение по сегментам** — таблица с цветовыми шкалами, сортируемая. Для "Не определено" — список компаний
3. **Характеристика сегментов** — текстовое описание каждого сегмента с примерами
4. **Какие признаки удалось извлечь** — сводка по всем признакам с количеством и долей
5. **Компании с признаками и без** — список, сколько компаний имеют хоть какие-то данные, сколько нет
6. **Признаки по сегментам** — таблица, сколько компаний в каждом сегменте содержат каждый признак
7. **Детальные признаки по каждому документу** — с фильтрацией по сегменту, кол-ву сотрудников, маршрутам, вместимости. Все столбцы сортируемые.

## Формат результатов

Все результаты в `results/` в формате CSV (UTF-8):
- `segmentation.csv` — сегмент, кол-во, доля, описание, список компаний
- `content_features.csv` — все извлечённые признаки по каждому юниту
- `feature_stats_total.csv` — общая статистика признаков
- `feature_stats_by_segment.csv` — статистика признаков в разрезе сегментов
- `feature_presence.csv` — список компаний с признаком "есть хоть какие-то данные / нет"
- `contract_durations.csv` — сроки контрактов (если извлечены)
- `report.html` — HTML-отчёт с фильтрацией и сортировкой

## Требования

- Python 3.8+
- Библиотеки: python-docx, pdfplumber, openpyxl
- Поддерживаемые форматы: .docx, .pdf, .xlsx (текстовые)
- .doc — не поддерживается (пропускается с предупреждением)
- .png/.jpg — пропускаются
